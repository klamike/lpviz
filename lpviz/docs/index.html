Standard Form

Each solver in lpviz has its own internal standard form. The form that is displayed in the frontend is:

$$\begin{align} \max_x \quad & c^\top x \\ \text{s.t.} \quad & Ax \leq b \end{align}$$

where:

  • $x \in \mathbb{R}^2$ is the vector of decision variables
  • $c \in \mathbb{R}^2$ is the vector of objective coefficients
  • $A \in \mathbb{R}^{m \times 2}$ is the constraint matrix
  • $b \in \mathbb{R}^m$ is the vector of right-hand side values
When using the interactive problem builder, the primal feasible region (the set $\{x \in \mathbb{R}^2 : Ax \leq b\}$) is non-empty and bounded.

Solvers

lpviz implements four solvers: IPM, PDHG, Simplex, and Central Path. Click on each solver below for detailed explanations with code references.

Simplex Algorithm

The simplex method implements a two-phase revised simplex algorithm with Bland's anti-cycling rule. The first phase finds an initial basic feasible solution and the second phase finds the optimal solution.

$$\begin{align} \text{Phase 1:} \quad & \min \enspace \sum_{i=1}^m t_i \quad \text{s.t.} \quad Ax + t = b, \quad x \geq 0, \quad t \geq 0 \\ \text{Phase 2:} \quad & \max \enspace c^\top x \quad\enspace \text{s.t.} \quad Ax = b, \quad x \geq 0 \end{align}$$

In 3D mode, the z-axis of each iterate of Simplex is set to $c^\top x$.

Interior Point Method (IPM)

IPM implements an infeasible primal-dual interior point algorithm. The core IPM routine requires problems be converted to the standard form $\min \enspace c^\top x \enspace \text{s.t.} \enspace Ax \geq b$. Each iteration involves solving two linear systems differing only in their right-hand sides.

$$\begin{bmatrix} A & -I & 0 \\ 0 & 0 & A^\top \\ 0 & Y & S \end{bmatrix} \begin{bmatrix} \Delta x \\ \Delta s \\ \Delta y \end{bmatrix} = \begin{cases}\text{predictor:}& \begin{bmatrix} b - Ax + s \\ c - A^\top y \\ -s \odot y \end{bmatrix} \\[1em] \text{corrector:}& \begin{bmatrix} 0 \\ 0 \\ \sigma \mu \mathbf{1} - \Delta s_{\text{aff}} \odot \Delta y_{\text{aff}} \end{bmatrix}\end{cases}$$

In 3D mode, the z-axis is of each iterate of IPM is set to $c^\top x + \frac{s^\top y}{m}$.

Primal-Dual Hybrid Gradient (PDHG)

PDHG implements a first-order method that alternates between primal and dual updates. Most textbook implementations first transform the problem to the equality standard form $Ax = b, x \geq 0$, then apply:

$$\begin{align} x^{k+1} &= \Pi_{\geq 0}(x^k - \eta(c + A^\top y^k)) \\ y^{k+1} &= y^k + \tau(A \tilde{x}^{k+1} - b) \end{align}$$

where $\tilde{x}^{k+1} = 2x^{k+1} - x^k$ is the extrapolated primal variable, $\Pi_{\geq 0}$ projects onto the non-negative orthant, and $\eta, \tau$ are step sizes.

In 3D mode, the z-axis of each iterate of PDHG is set to $c^\top x$.

PDHG Inequality Mode

The inequality mode avoids transforming the problem, treating the inequalities directly. It is similar to applying PDHG to the dual problem:

$$\begin{align} y^{k+1} &= \Pi_{\geq 0}(y^k + \tau(A x^k - b)) \\ x^{k+1} &= x^k - \eta(c + A^\top \tilde{y}^{k+1}) \end{align}$$

where $\tilde{y}^{k+1} = 2y^{k+1} - y^k$ is the extrapolated dual variable. Due to the inequality constraints in the primal, the dual variables $y$ are now constrained to be non-negative.

Central Path

The central path is a smooth curve of solutions to the barrier problem as $\mu \to 0$. Each point on the path is found using Newton's method with Armijo line search.

$$x(\mu) = \arg\min_x \left( c^\top x - \mu \sum_{i=1}^m \log(b_i - a_i^\top x) \right)$$

In 3D mode, the z-axis is of each iterate of Central Path is set to $c^\top x(\mu) - \mu \sum_{i=1}^m \log(b_i - a_i^\top x(\mu))$.
